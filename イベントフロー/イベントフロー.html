イベントフロー(Scorpio)

ユースケース名 アカウント情報の登録をする
概要 アカウント情報を登録し、社内SNSを利用できるようにする。
アクター 未登録ユーザ
事前条件 登録・ログイン画面が表示されていること。
事後条件 アカウント情報の登録が完了し、メイン画面に移動していること。
基本フロー 1. 未登録ユーザは登録・ログイン画面から登録画面への移動を指示する。
2. システムは登録画面を表示する。
3. 未登録ユーザは氏名・メールアドレス・パスワードを入力し、登録確認画面への移動を指示する。
4. システムは登録確認画面を表示する。
5. 未登録ユーザは内容を確定する。
6. システムはメイン画面を表示する。
ユースケース名 アカウント情報の変更をする
概要 登録済のアカウント情報を変更し反映させる。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。
事後条件 変更後のアカウント情報がシステムに保存され、メイン画面を表示していること。
基本フロー 1. 登録済ユーザはメイン画面から変更画面への移動を指示する。
2. システムは変更画面を表示する。
3. 登録済ユーザは変更したいアカウント情報を入力し変更確認画面への移動を指示する。
4. システムは変更確認画面を表示する。
5. 登録済ユーザは内容を確定する。
6. システムはメイン画面を表示する。
ユースケース名 アカウント情報を検索する
概要 検索したいアカウントの氏名を入力し、検索結果を表示する。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。
事後条件 検索結果が表示されていること。
基本フロー 1. 登録済ユーザはメイン画面から検索画面への移動を指示する。
2. システムは検索画面を表示する。
3. 登録済ユーザは検索フォームに検索したいユーザの氏名を入力する。
4. システムは検索結果画面を表示し、入力内容に該当する検索結果を表示する。
ユースケース名 日記を投稿する
概要 日記をシステムへ投稿し、友達などが閲覧できるようにする。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。
事後条件 日記の投稿内容がシステムに保存されていること。タイムライン画面が表示されていること。
基本フロー 1. 登録済ユーザはメイン画面から投稿画面への移動を指示する。
2. システムは投稿画面を表示する。
3. 登録済ユーザは日記の投稿内容を入力し公開範囲を選択する。
4. 登録済ユーザは入力内容を確定する。
5. システムはタイムライン画面を表示する。
ユースケース名 友達の日記にコメントする
概要 ユーザは友達の日記にコメントし反映させる。
アクター 登録済ユーザ
事前条件 友達の日記が投稿されていること。登録済ユーザはログイン済であること。
事後条件 コメントがシステムに保存されていること。
基本フロー 1. 登録済ユーザはタイムライン画面から日記の詳細画面への移動を指示する。
2. システムは日記の詳細画面を表示する。
3. 登録済ユーザはコメントの入力画面への移動を指示する。
4. システムはコメントの入力画面を表示する。
5. 登録済ユーザはコメントの投稿内容を入力する。
6. 登録済ユーザは入力内容を確定する。
7. システムはコメントに日記と同じ公開範囲を設定し反映させる。
8. システムは日記の詳細画面を表示する。
ユースケース名 友達の日記にリアクションをする
概要 ユーザは友達の日記にリアクションをし反映させる。
アクター 登録済ユーザ
事前条件 友達の日記が投稿されていること。登録済ユーザはログイン済であること。
事後条件 リアクションの状態がシステムに保存されていること。
基本フロー 1. 登録済ユーザはタイムライン画面から日記の詳細画面への移動を指示する。
2. システムは日記の詳細画面を表示する。
3. 登録済ユーザはリアクションの状態を変更する。
4. システムはリアクションの状態を反映させる。
ユースケース名 友達の申請をする
概要 登録済ユーザは他の登録済ユーザに友達の申請を送る。
アクター 登録済ユーザ
事前条件 他の登録済ユーザが登録済ユーザからの友達申請の待機中ではないこと。登録済ユーザはログイン済であること。
事後条件 他の登録済ユーザに申請が送られ、待機中が表示されていること。
基本フロー 1. 登録済ユーザは検索結果画面から他の登録済ユーザのプロフィール画面への移動を指示する。
2. システムはプロフィール画面を表示する。
3. 登録済ユーザは他の登録済ユーザに申請を送る。
4. システムは申請を反映させる。
ユースケース名 友達申請の承認をする
概要 他の登録済ユーザから来た申請を承認する。
アクター 登録済ユーザ
事前条件 他の登録済ユーザから申請が送られていること。登録済ユーザはログイン済であること。
事後条件 他の登録済ユーザに承認が反映されていること。
基本フロー 1. 登録済ユーザはメイン画面から申請確認画面への移動を指示する。
2. システムは申請確認画面を表示する。
3. 登録済ユーザは他の登録済ユーザからの申請を承認する。
4. システムは承認を反映させる。
ユースケース名 ログインする
概要 アカウントにログインする。
アクター 登録済ユーザ
事前条件 登録・ログイン画面が表示されていること。
事後条件 ログインが完了し、メイン画面に移動していること。
基本フロー 1. 登録済ユーザは登録・ログイン画面からログイン画面への移動を指示する。
2. システムはログイン画面を表示する。
3. 登録済ユーザはアカウント情報を入力し、ログイン確認画面への移動を指示する。
4. システムはログイン確認画面を表示する。
5. 登録済ユーザは入力内容を確認し確定する。
6. システムはメイン画面を表示する。
ユースケース名 タイムラインを表示する
概要 投稿された日記をタイムラインに表示する。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。
事後条件 タイムライン画面が表示されていること。
基本フロー 1. 登録済ユーザはメイン画面からタイムライン画面への移動を指示する。
2. システムはタイムライン画面を表示し、タイムライン上に閲覧範囲が全体の日記を一覧表示する。
3. 登録済ユーザは必要があれば閲覧範囲を変更する。
4. システムは指定された閲覧範囲の日記を一覧表示する。
ユースケース名 日記の詳細を表示する
概要 投稿された日記の詳細を表示する。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。タイムライン画面に日記の投稿があること。
事後条件 日記の詳細が表示されていること。
基本フロー 1. 登録済ユーザはタイムライン画面から日記の詳細画面への移動を指示する。
2. システムは日記の詳細画面を表示する。
ユースケース名 ログアウトする
概要 アカウントからログアウトする。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。
事後条件 ログアウトが完了し、登録・ログイン画面に移動していること。
基本フロー 1. 登録済ユーザはメイン画面からログアウトを指示する。
2. システムはログアウト確認通知を表示する。
3. 登録済ユーザはログアウト確認通知からログアウト確認を行う。
4. システムはログアウト処理を行い、登録・ログイン画面に移動する。
ユースケース名 通知を表示する
概要 通知一覧画面を表示し、通知を確認する。
アクター 登録済ユーザ
事前条件 登録済ユーザはログイン済であること。
事後条件 通知一覧画面が表示されていること。
基本フロー 1. 登録済ユーザはメイン画面から通知一覧画面への移動を指示する。
2. システムは通知一覧画面に移動し、通知を表示する。
